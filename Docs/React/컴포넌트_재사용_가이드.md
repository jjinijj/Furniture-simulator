# React 컴포넌트 재사용 설계 문서 (Portfolio)

이 문서는 **React + TypeScript(TSX)** 기반 프로젝트에서
컴포넌트와 스타일을 어떻게 **재사용 가능하고 확장성 있게 설계했는지**를 설명하기 위한 포트폴리오 문서입니다.

본 문서의 목적은 단순 사용법 나열이 아닌,
**설계 의도, 판단 기준, 실무 적용 가능성**을 보여주는 데 있습니다.

---

## 1. 설계 목표

- UI 중복 제거 및 유지보수 비용 최소화
- 컴포넌트 단위 책임 분리
- 스타일 충돌 방지
- 규모 확장 시에도 구조가 무너지지 않도록 설계

---

## 2. 컴포넌트 재사용 기준

다음 기준 중 하나라도 충족하면 재사용 컴포넌트로 분리했습니다.

- 동일한 UI 패턴이 **2회 이상** 등장
- 다른 화면/기능에서도 사용 가능
- props를 통해 내용 또는 동작이 변경 가능

> ❝ 복사하지 않고 조합한다 (Composition over Copy) ❞

---

## 3. 컴포넌트 설계 전략

### 3.1 Presentational Component (UI 전용)

```tsx
interface BoxProps {
  children: React.ReactNode;
}

export function Box({ children }: BoxProps) {
  return <div className={styles.box}>{children}</div>;
}
```

- 상태 및 비즈니스 로직 없음
- 오직 UI 표현에만 집중
- 다양한 컴포넌트에서 감싸는 용도로 재사용

---

### 3.2 Container Component (로직 담당)

```tsx
export function UserBox() {
  const user = useUser();
  return <Box>{user.name}</Box>;
}
```

- 데이터 로딩, 상태 관리 담당
- UI 컴포넌트 조합

이 구조를 통해 **UI와 로직의 결합도를 최소화**했습니다.

---

## 4. 스타일 재사용 설계

### 4.1 컴포넌트 전용 스타일

```txt
InfoBox/
 ├─ InfoBox.tsx
 └─ InfoBox.module.css
```

- 해당 컴포넌트에서만 사용
- CSS Module을 사용하여 scope 격리

---

### 4.2 공용 스타일 분리 (여러 컴포넌트에서 사용)

```txt
styles/
 └─ common.module.css
```

```css
.box {
  padding: 30px;
  border-radius: 12px;
  background-color: #1e1e1e;
}
```

```tsx
import common from '@/styles/common.module.css';

<div className={common.box} />
```

- 반복되는 UI 패턴을 공용 스타일로 정의
- 전역 오염 없이 재사용 가능

---

### 4.3 조합형 스타일 전략

```css
.paddingLg { padding: 30px; }
.rounded { border-radius: 12px; }
```

```tsx
<div className={`${common.paddingLg} ${common.rounded}`} />
```

- 필요에 따라 스타일을 조합
- 디자인 시스템 확장에 유리

---

## 5. Props를 활용한 확장성

```tsx
interface CardProps {
  title: string;
  highlighted?: boolean;
}

export function Card({ title, highlighted }: CardProps) {
  return (
    <div className={highlighted ? styles.highlight : styles.normal}>
      {title}
    </div>
  );
}
```

- props를 통해 UI 상태 표현
- 조건 분기를 최소화하여 가독성 유지

---

## 6. 안티 패턴과 개선 이유

| 안티 패턴 | 문제점 | 개선 방식 |
|---------|-------|-----------|
| 스타일을 App.css에 몰아넣기 | 전역 충돌, 관리 어려움 | CSS Module 분리 |
| 유사 컴포넌트 복사 | 중복 증가 | 공통 컴포넌트 추출 |
| 하드코딩된 UI | 재사용 불가 | props 기반 설계 |

## 8. 정리

- 컴포넌트는 작게, 책임은 명확하게
- 스타일은 scope를 가지게 분리
- 재사용은 복사가 아닌 설계로 해결

본 구조는 **포트폴리오 및 실무 프로젝트 모두에 적용 가능한 패턴**을 목표로 설계되었습니다.
